## ניתוח מסמך זדוני באמצעות oledump.py

במשימה זו נשתמש בכלי **oledump.py** כדי לבצע ניתוח סטטי על מסמך Excel שחשוד כזדוני.

**Oledump.py** הוא כלי מבוסס Python המנתח קובצי OLE2 (המכונים לעיתים Structured Storage או Compound File Binary Format). ראשי התיבות OLE מייצגים את הטכנולוגיה **Object Linking and Embedding** של מיקרוסופט. קובצי OLE2 משמשים בדרך כלל לאחסון סוגי נתונים מרובים, כגון מסמכים, גיליונות אלקטרוניים ומצגות, בתוך קובץ יחיד. כלי זה שימושי מאוד לחילוץ ובחינת התוכן של קובצי OLE2, מה שהופך אותו למשאב בעל ערך עבור ניתוח פורנזי וזיהוי נוזקות.

## **יומן אנליסט: ניתוח קובץ Agent Tesla**

היום ניתחתי קובץ Excel חשוד בשם `agenttesla.xlsm` בתוך סביבת ה-REMnux שלי. המטרה הייתה להבין מה הקובץ הזה באמת מנסה לעשות בלי להריץ אותו בטעות.

### **1. זיהוי ה"אקדח המעשן"**

התחלתי בהרצת `oledump.py`. מיד זיהיתי שבסטרים מספר 4 מופיעה האות **'M'** גדולה. זה היה הרגע שבו הבנתי שלא מדובר בסתם גיליון נתונים, אלא במסמך שמכיל **מאקרו (VBA)**. תוקפים מתים על המקומות האלו כי הם רצים אוטומטית כשהמשתמש מאשר "Enable Content".

### **2. פריסת הקוד המוסתר**

ניסיתי לקרוא את הסטרים, אבל הוא היה דחוס. השתמשתי בפרמטר `--vbadecompress` כדי לראות את הקוד המקורי. מה שראיתי שם היה קוד VBA "מלוכלך" (Obfuscated) מלא בתווים כמו `^` ו-`*`. התוקף ניסה לבלבל אותי ואת מערכות האבטחה, אבל זה רק גרם לי להיות יותר סקרן.

### **3. הניקוי ב-CyberChef**

לקחתי את הפקודה המלוכלכת ל-CyberChef. בעזרת פעולות Find/Replace פשוטות, ניקיתי את כל ה"רעש" הויזואלי. פתאום הכל נהיה ברור: פקודת **PowerShell** שרצה ברקע (`WindowStyle hidden`) ועוקפת את הגדרות המערכת (`executionpolicy bypass`).

### **4. המסקנה הסופית**

הבנתי שהקובץ הזה הוא בסך הכל "שער כניסה" (Downloader). המאקרו פונה לכתובת IP ציבורית (`193.203.203.67`), מוריד קובץ שמתחזה ל-PDF אבל הוא בעצם קובץ הרצה זדוני (`.exe`), ושומר אותו בתיקייה זמנית במחשב. ברגע שההורדה מסתיימת, הוא פשוט מריץ אותו.
